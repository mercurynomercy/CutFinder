# CutFinder 手动测试清单 (Manual Test Checklist)

> 个人验收用。逐项勾选，确认 UI 与所有功能端到端可用。
> 标记说明：**[UI]** 纯界面、无需模型；**[OMLX]** 需 OMLX + 真实素材；**[约束]** 不可破坏原素材的硬性约束。
>
> 准备一个**测试素材库**（空目录）和一个含 **A-roll（有中文解说）+ B-roll（纯空镜）** 的素材文件夹再开始。

---

## 0. 前置准备 (Prerequisites)

- [ ] OMLX 已启动，且已加载 `Qwen3.6-35B-A3B`（文本）与 `Qwen3-VL-8B`（视觉）
- [ ] `make check-omlx` → 输出 `OMLX OK` 且必需模型齐全
- [ ] `ffmpeg -version` 与 `ffprobe -version` 可用
- [ ] 根目录 `.env` 已填 `OMLX_BASE_URL` / `OMLX_API_KEY`
- [ ] `make dev` 启动成功；后端 http://localhost:5081（FastAPI）、前端 http://localhost:5080（Vite）都能打开

## 1. 自动化测试 & 构建 (先跑一遍做基线)

- [ ] `make test-unit` → **330 passed**（纯单元测试，无外部依赖）
- [ ] `cd frontend && npm test` → **174 passed**（Vitest 单元/组件测试）
- [ ] `cd frontend && npm run build` → **退出码 0**（`tsc -b` 类型检查干净 + Vite 构建产物生成）
- [ ] `cd backend && uv run mypy cutfinder/` → `Success: no issues`（strict 模式）
- [ ] `cd backend && uv run ruff check cutfinder/` → `All checks passed`

---

## 2. 库绑定 & 设置页 [UI]

- [ ] **未绑定库**时：打开 5080 显示「设置 / 绑定向导」；目录类接口返回 503（不崩溃）
- [ ] 设置页点 **Choose…** → 弹出 **macOS 原生文件夹选择框**（不是浏览器选择）
- [ ] 选一个目录 → 输入框填入的是**绝对路径**（如 `/Users/you/Library`），不是文件夹名
- [ ] 绑定库 → **无需重启**即热生效，缩略图墙可访问
- [ ] **Add folder**（素材文件夹）同样用原生选择框，存的是绝对路径
- [ ] 移除一个素材文件夹 → 列表更新
- [ ] 库路径设定后显示「不可更改」提示
- [ ] 可编辑：文本模型 / 视觉模型 / Whisper 模型 / 扩展名白名单 / B-roll 抽帧数 / VAD 阈值
- [ ] 输出语言可切换 **中文 / 英文**
- [ ] 保存设置 → 刷新页面/重开后仍保留（持久化到 `~/.cutfinder` 与 `<库>/.cutfinder/config.json`）

## 3. 扫描 & 实时进度 [OMLX]

- [ ] 点 **Scan** → 顶部出现进度条，任务开始
- [ ] 逐片进度通过 SSE 实时更新（当前处理的文件名/阶段）
- [ ] 任务完成 → 出现完成提示，缩略图墙填充出片段
- [ ] **幂等重扫**：再点一次 Scan → 已处理的文件按指纹去重，**不重复复制**、不重复入库；只处理新文件
- [ ] 源文件夹为空 / 无匹配扩展名（`.mov .mp4 .m4v`）→ 任务正常结束（total=0），不报错
- [ ] 源文件夹含非白名单扩展名文件（如 `.avi`、`.mxf`）→ 自动跳过，不入库

## 4. 任务队列页 [UI]（需先跑过/正在跑任务）

- [ ] 顶部入口按钮打开「任务队列」全屏页
- [ ] 列表显示每个任务：类型（扫描/重新分析）、状态徽章、进度 `done/total`、失败数、开始时间
- [ ] 状态徽章中文正确：排队中 / 进行中 / 已完成 / 失败 / 已取消
- [ ] **删除**一个已完成任务 → 该行消失
- [ ] **删除**一个进行中/排队中的任务 → 先取消再删除（worker 停止处理它）
- [ ] **重试失败项**按钮只在 `failed > 0` 时出现 → 点击后只把失败的片段重新入队
- [ ] **暂停** → worker 不再领新任务（当前这片跑完才停）；**恢复** → 继续，队列不丢
- [ ] 无任务时显示空状态「暂无任务」

## 5. A-roll 流水线 [OMLX]

- [ ] 有中文解说的片段被判为 **A-roll**
- [ ] 详情里能看到**转写全文**（whisper）
- [ ] 生成了**中文简介**
- [ ] 生成了**标签**
- [ ] 副本落在 `<库>/YYYY-MM-DD/A-roll/`

## 6. B-roll 流水线 [OMLX]

- [ ] 纯空镜（无人声）片段被判为 **B-roll**
- [ ] 生成了**画面描述**（视觉模型）
- [ ] 生成了**画面标签**
- [ ] 副本落在 `<库>/YYYY-MM-DD/B-roll/`

## 7. 归档 & 原素材约束 [约束]（务必逐项确认）

- [ ] 副本按 `<库>/YYYY-MM-DD/A-roll(或 B-roll)/` 组织
- [ ] **原文件只读**：扫描后源文件夹里的原文件**内容/修改时间未变**；源文件夹里**没有**遗留 `_frame_*.png` 等临时帧（应只在系统临时目录）
- [ ] **拍摄时间不变**：副本保留文件系统时间（`shutil.copy2`）；分类日期取自内嵌 EXIF/QuickTime 时间，缺失时回退文件创建时间并在 UI 标注 `date_source`
- [ ] **离线**：整个流程无外发网络请求（素材不出本机）
- [ ] **幂等**：重复扫描不产生重复副本

## 8. 缩略图墙 & 检索 [UI]

- [ ] 缩略图正常渲染（无图时显示占位图标）
- [ ] 按**日期**筛选
- [ ] 按 **A-roll / B-roll** 类型筛选
- [ ] 按**标签**筛选
- [ ] **全文搜索**：按台词/简介/描述关键词能搜到对应片段
- [ ] 状态为 `partial` 的片段缩略图左下角显示琥珀色「**部分**」标记（hover 提示「AI 分析未完成，可重新分析」）

## 9. 详情面板（右侧抽屉）[UI/OMLX]

- [ ] 点缩略图 → 抽屉打开，显示元数据（分辨率 / fps / 编码 / 时长 / 拍摄时间 / date_source）
- [ ] A-roll：显示简介 + 转写；B-roll：显示画面描述
- [ ] 编辑简介/描述 → 保存生效
- [ ] 编辑标签（增/删）→ 保存生效；**手动标签**在重分析后保留
- [ ] 点 **A-roll / B-roll** 纠正分类 → 类型改变，记为手动（不会被下次自动扫描覆盖）
- [ ] **Re-analyze** → 显示「Re-analyzing…」，跑完**自动刷新**出新简介/描述/标签
- [ ] **→ B-roll & re-analyze**（一键纠正+重跑）→ 切到另一类型并走对应管道重跑，结果刷新
- [ ] 重分析后**手动纠正的 A/B 与手动标签仍保留**
- [ ] `Esc` 或点抽屉外 → 关闭

## 10. AI 输出语言 [OMLX]

- [ ] 设置 `输出语言 = 英文` → 之后新分析/重分析的简介/描述为**英文**
- [ ] 改回中文 → 新分析为**中文**

## 11. 错误 & 边缘情况

- [ ] **AI 分析失败的片段仍被归档**：原文件按日期+类型复制进库，状态 `partial`，缩略图有「部分」标记（不是直接丢弃）[约束]
- [ ] **被误分类**的片段（如餐厅场景被判 A-roll，转写是噪声）：用「→ B-roll & re-analyze」纠正后能拿到正常画面描述 [OMLX]
- [ ] **OMLX 关闭/不可达**时：扫描任务不挂死（有 `max_tokens` 兜底），片段进入 `partial/error`，可在恢复后重试
- [ ] 缩略图接口可用：浏览器直接访问 `http://localhost:5081/api/clips/<id>/thumbnail` 能返回图片
