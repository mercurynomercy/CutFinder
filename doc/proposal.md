# CutFinder 需求文档（Proposal）

> 一个本地运行的视频素材（footage）智能分类与检索工具，灵感来自 [Argus](https://github.com/discoposse/argus)。
> 面向个人 Vlog 创作者：自动区分 A-roll / B-roll、为 A-roll 生成中文简介、按日期与类型分类、打标签、生成缩略图，方便日后快速找回素材。

- **版本**：v1（需求文档）
- **日期**：2026-06-13
- **平台**：macOS（Apple Silicon），配合 Final Cut Pro 工作流
- **运行形态**：本地 Web App（浏览器打开 `localhost`），全程离线，AI 全本地
- **详细设计**：模块划分、接口、数据模型、测试与部署见 [`doc/detailed-design.md`](./detailed-design.md)

---

## 1. 背景与目标

我有大量日常 Vlog 素材，分两类：

- **A-roll**：有人出镜/解说的镜头，通常带**中文解说**。
- **B-roll**：纯画面、无解说的空镜。

素材越积越多，靠人工找内容很慢。本工具的目标：**自动分析、分类、打标签、生成缩略图与简介**，让我之后能按日期 / 类型 / 标签 / 台词快速定位到任意一段素材。

### 核心设计原则

1. **原文件只读，绝不破坏**：所有整理动作只发生在「复制出来的新素材库」里。
2. **拍摄时间永不改变**：视频内嵌的 QuickTime / EXIF 拍摄时间全程不被修改，复制时同时保留文件系统的修改时间（`shutil.copy2`）。
3. **全本地、离线**：不上传任何素材到云端，所有 AI 推理在本机完成。
4. **幂等可重跑**：再次扫描只处理新文件，已处理的按文件指纹跳过，不重复复制。

## 3. 技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 运行形态 | 本地 Web App（浏览器访问 `localhost`） | AI/视频工具链最顺；之后可用 Tauri / PyInstaller 打包成 `.app` |
| 模型服务 | [OMLX](https://github.com/jundot/omlx)（本地推理服务器，OpenAI 兼容） | 同一个接口同时托管文本与视觉模型，内置内存管理 |
| 后端 | Python + FastAPI | 视频与 AI 生态最全 |
| 数据库 | SQLite（单文件，存于素材库旁） | 零配置、易备份、便于迁移 |
| 视频处理 | ffmpeg / ffprobe | 抽帧、缩略图、读取元数据（拍摄时间、时长、分辨率、编码） |
| 前端 | Vite + React + Tailwind + shadcn/ui | 缩略图墙 + 多维筛选/搜索；深色 UI 见 [`ui-design.md`](./ui-design.md) |
| 进度反馈 | 后台任务队列 + SSE | 批量处理时实时推送逐个进度 |
| 测试 | pytest + Vitest/RTL + 少量 Playwright | 外部依赖接口抽象、可独立测试 |
| 环境/部署 | mise + uv + Brewfile + Makefile + 全局配置（`~/.cutfinder/config.json`） | 一键安装，换机免折腾（Docker 不适用：MLX 需 Metal） |

### AI 模型（全本地）

| 用途 | 模型 | 运行方式 |
|------|------|---------|
| 人声检测（A/B 判定） | Silero VAD（轻量） | 本地推理（独立） |
| 人声分离（转写前去 BGM） | **Demucs**（`htdemucs`，约 80MB） | 本地推理（torch/MPS，独立）；抽干声扔伴奏后再转写 |
| 中文语音转写（A-roll） | `mlx-whisper`（large-v3） | 本地独立进程（OMLX 不托管音频） |
| A-roll 简介 + 标签（文本总结） | **Qwen3.6-35B-A3B**（文本 MoE，用户现有） | OMLX（`localhost:8000/v1`，`/chat/completions`） |
| B-roll 画面识别 + 标签 | **Qwen3-VL-8B-Instruct**（视觉，默认 4/6-bit） | OMLX（同一接口，`model` 名切换，发 base64 帧） |

> **模型分工说明**：`Qwen3.6-35B-A3B` 是**文本**模型，看不了画面，只做文字总结；`Qwen3-VL-8B-Instruct` 是**视觉**模型，专门「看」B-roll 抽帧。两者**由 OMLX 同时托管**，通过同一个 OpenAI 兼容接口、用不同 `model` 名调用；视觉输入按 OpenAI 标准用 base64 图片传帧。OMLX 负责模型的加载/卸载与内存管理（LRU 淘汰、pin、按模型 TTL）。
>
> **例外**：Whisper 语音转写 OMLX 不托管，由本地独立的 `mlx-whisper` 进程完成。模型与量化等级在 OMLX 与本工具设置中均可替换。
>
> **转写前人声分离（去 BGM，见 [`tasks/18`](./tasks/18-vocal-separation.md)）**：后期混入的背景音乐会被 whisper 一并转成文字或诱发幻觉。转写前用 **Demucs** 抽出人声干声、扔掉伴奏，再喂 whisper。**字幕导出（成片）强制分离**；**A-roll 入库流水线提供开关 `vocal_separation`，默认关闭**（原始素材通常无后期配乐）。分离失败时回落原始音频、转写不中断。

## 4. 处理流水线

```
用户配置：[源文件夹] + [目标素材库文件夹]
        │
        ▼
  扫描视频文件（.mov/.mp4/.m4v 等）
  └─ 按文件指纹去重，已处理的跳过
        │
        ▼
  ffprobe 读元数据（拍摄日期/时长/分辨率/编码）
  ffmpeg 生成缩略图（代表帧）
        │
        ▼
  ┌────── Silero VAD：检测是否有人声解说 ──────┐
  │                                            │
 有人声 (A-roll)                          无人声 (B-roll)
  ▼                                            ▼
mlx-whisper 转写中文全文(独立)          ffmpeg 抽若干关键帧
  ▼                                            ▼
Qwen3.6 生成「简介+标签」(OMLX)         Qwen3-VL 给抽帧打「标签+描述」(OMLX·base64)
  │                                            │
  └─────────────────┬──────────────────────────┘
                    ▼
   写入 SQLite（元数据/类型/简介/标签/转写全文/缩略图路径）
                    ▼
   复制原文件到：  库/YYYY-MM-DD/A-roll(或 B-roll)/原文件名
   （保留内嵌拍摄时间与文件时间，原文件不动）
```

## 5. 分类、标签与浏览

### 5.1 分类目录结构

```
<素材库>/
├── 2026-06-13/
│   ├── A-roll/
│   │   └── clip_001.mov
│   └── B-roll/
│       └── clip_002.mov
├── 2026-06-14/
│   └── ...
├── .cutfinder/
│   ├── catalog.sqlite        # 数据库
│   └── thumbnails/           # 缩略图
```

- 日期取自视频**内嵌拍摄时间**（无内嵌时间则回退到文件创建时间，并在 UI 标注「日期来源不确定」）。
- 同名冲突时自动追加序号，不覆盖。

### 5.2 标签

- **来源**：AI 自动生成（A-roll 来自转写内容；B-roll 来自画面识别）+ 用户手动增删。
- 自动与手动标签都入库，UI 可区分来源。

### 5.3 浏览与检索（前端）

- **缩略图墙**：每段素材一张代表帧。
- **多维筛选**：按 日期 / A-B-roll / 标签。
- **全文搜索**：A-roll 可按转写**台词全文**搜索；B-roll 可按画面描述/标签搜索。
- **详情**：单段素材显示简介、标签（可编辑）、转写全文、元数据、库内路径。
- **重新分析**：单段素材可一键 re-analyze（换模型或结果不佳时重跑 AI），保留手动纠正与手动标签，不重新复制原文件。

## 6. 配置项

配置分两类来源：**密钥/端点走全局配置或 OS env**（不进 git），**用户偏好走 JSON**（`<库>/.cutfinder/config.json`）。

**全局配置 / OS env vars（密钥/端点）**
- `OMLX_BASE_URL`（默认 `http://localhost:8000/v1`）—— OMLX 接口，可设在 `~/.cutfinder/config.json` 或 OS env。
- `OMLX_API_KEY`（必填）—— OMLX 鉴权，不设默认值；缺失时启动报错。

**JSON（用户偏好）**
- 源文件夹（可多个）、目标素材库文件夹。
- **文本模型名**（默认 `Qwen3.6-35B-A3B`）—— 用于 A-roll 简介/标签。
- **视觉模型名**（默认 `Qwen3-VL-8B-Instruct`，4/6-bit）—— 用于 B-roll 画面识别；量化等级在 OMLX 侧选择。
- Whisper 模型档位（默认 `large-v3`，独立于 OMLX）。
- 文件扩展名白名单（默认 `.mov .mp4 .m4v`）。
- B-roll 抽帧数量（默认 `3`）、VAD 阈值（默认 `0.15`）。
- **A-roll 转写前人声分离开关** `vocal_separation`（默认 `false`）—— 开启后之后 scan 的新 A-roll 先用 Demucs 去 BGM 再转写。字幕导出强制分离，不受此开关影响。

## 7. v1 范围边界

### v1 包含

需求 0–7 全部：自定义文件夹、保留拍摄时间、A/B 自动判定（可纠正）、A-roll 中文简介、按日期+类型自动分类、标签（自动+手动）、缩略图、接本地 Qwen3.6。

### v1 不包含（后续版本）

- **需求 8：关键帧建议** —— 复用 Qwen3-VL 视觉模型，为每段素材推荐最佳剪辑切点/代表帧。
- **Final Cut Pro 深度集成** —— 导出 FCPXML / 关键词（Keywords / Smart Collections），把分类与标签直接灌入 FCP 资源库。
- 打包为独立 `.app`（Tauri / PyInstaller）。

## 8. 关键风险与对策

| 风险 | 对策 |
|------|------|
| 复制改动原文件 | 原文件全程只读；只对副本操作；复制后校验大小/指纹 |
| 拍摄时间被改 | 不改原文件；副本用 `copy2` 保留时间；内嵌元数据天然不变 |
| 占用双倍磁盘空间（复制模式） | 文档明示；后续可提供「仅索引/移动」模式作为选项 |
| AI 判定/简介出错 | A/B 可手动纠正并记住；简介与标签均可人工编辑 |
| 文本+视觉两模型同驻内存 | 交由 OMLX 内存管理（LRU/pin/TTL）；视觉默认 4/6-bit 量化；抽帧数量可配置 |
| 中文转写准确率 | 采用 Whisper large-v3（中文表现好）；保留全文供人工核对 |
